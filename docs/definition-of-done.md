# KWANUS OS — Definition of Done

A task is considered DONE when:

## Code Quality
- Code compiles without errors
- No console errors in browser
- No TypeScript errors
- No unused mock data

## Backend
- API routes return correct JSON shapes
- Prisma models match frontend expectations
- Migrations applied successfully
- Services handle errors gracefully

## Frontend
- UI displays real data
- Loading and error states implemented
- No placeholder or mock values

## Testing
- Manual test completed
- End-to-end flow validated (Upload → Analyze → Dispute)
- No 500 errors in network tab

## Documentation
- PR includes summary + screenshots
- Linked to correct issue and milestone

## Review
- Approved by reviewer
- Merged into main without conflicts
